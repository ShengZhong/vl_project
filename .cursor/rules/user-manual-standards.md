**字段详细说明**：

#### 账户名称

| 项目 | 内容 |
|------|------|

- ⚠️ 带红色星号（*）的字段为必填项
- ⚠️ 表单会实时校验，错误信息会显示在字段下方
- 💡 系统会每 30 秒自动保存草稿，避免数据丢失

---

### 步骤 4: 提交表单

**操作说明**：

1. 确认所填信息无误后，点击对话框底部的 **提交** 按钮
2. 系统将进行数据校验和处理，请稍等片刻（通常 1-2 秒）

**截图占位符**：

\`\`\`
[截图4: 表单提交按钮和加载状态]
文件名: AD-ACC-001_step4_submit_button.png
说明: 展示提交按钮位置和加载状态（按钮变为"提交中..."）
\`\`\`

**可能的结果**：

#### ✅ 提交成功

| 项目 | 说明 |
|------|------|
| **系统提示** | 页面顶部显示绿色提示框："账户创建成功" |
| **页面变化** | 新建弹框自动关闭<br>账户列表自动刷新<br>新创建的账户显示在列表顶部（高亮显示 3 秒）|
| **邮件通知** | 如果填写了邮箱，将收到账户创建成功的邮件 |

#### ❌ 提交失败

| 错误提示 | 原因分析 | 解决方法 |
|---------|---------|---------|
| "账户名称已存在" | 该名称已被使用 | 更换一个唯一的账户名称 |
| "邮箱格式不正确" | 邮箱格式错误 | 检查邮箱格式（如 user@domain.com）|
| "手机号格式不正确" | 手机号不是 11 位 | 检查手机号码格式 |
| "权限不足" | 没有创建账户的权限 | 联系管理员分配相应权限 |
| "网络异常，请稍后重试" | 网络连接问题 | 检查网络连接，稍后重试 |
| "服务器繁忙，请稍后重试" | 服务器负载过高 | 等待 1-2 分钟后重试 |

**处理失败的步骤**：

1. 仔细阅读错误提示信息
2. 根据提示修改对应字段
3. 点击 **提交** 按钮重新提交
4. 如果多次失败，请联系技术支持

**注意事项**：

- ⚠️ 提交后请勿重复点击按钮，避免重复提交
- ⚠️ 如果超过 30 秒没有响应，请刷新页面后重试
- 💡 提交失败时，您填写的数据会保留，无需重新填写

### 3.7 常见问题 (FAQ)

```markdown
## ❓ 常见问题 (FAQ)

### Q1: 为什么我看不到"新建账户"按钮？

**A**: 可能的原因：

| 原因 | 解决方法 |
|------|---------|
| **权限不足** | 您的账号没有账户创建权限<br>→ 联系管理员分配权限 |
| **浏览器兼容性** | 使用了不支持的浏览器<br>→ 使用 Chrome、Safari、Edge 等现代浏览器 |
| **浏览器缓存** | 页面未正确加载<br>→ 按 Ctrl+F5 (Windows) 或 Cmd+Shift+R (Mac) 强制刷新 |

### Q2: 提交后长时间没有响应怎么办？

**A**: 请按以下步骤排查：

| 步骤 | 操作说明 |
|------|---------|
| **Step 1** | 检查网络连接：确保网络正常 |
| **Step 2** | 查看浏览器控制台：<br>按 F12 打开开发者工具<br>查看 Console 面板是否有错误信息 |
| **Step 3** | 刷新页面重试：<br>如果超过 30 秒仍无响应<br>刷新页面后重新操作 |
| **Step 4** | 联系技术支持：<br>如果问题持续存在<br>记录操作步骤和错误信息<br>联系技术支持解决 |

### Q3: 账户名称重复了怎么办？

**A**: 账户名称必须唯一，解决方法：

| 方法 | 说明 |
|------|------|
| **修改名称** | 在现有名称后添加序号或日期<br>例如：`test_account` → `test_account_20250115` |
| **使用命名规则** | 建议使用统一的命名规则<br>格式：`客户名_业务类型_日期`<br>例如：`zhangsan_ads_20250115` |
| **检查已有账户** | 先搜索是否已创建过<br>避免重复创建 |

### Q4: 如何批量操作多个账户？

**A**: 使用批量操作功能：

| 步骤 | 操作说明 |
|------|---------|
| **Step 1** | 勾选列表中需要操作的账户（支持全选）|
| **Step 2** | 点击列表上方的批量操作按钮：<br>- **批量启用**：启用选中的账户<br>- **批量禁用**：禁用选中的账户<br>- **批量导出**：导出选中的账户信息 |
| **Step 3** | 确认操作后系统将批量处理 |
| **Step 4** | 处理完成后显示成功数量和失败数量 |

**限制**：

- 单次批量操作最多 100 条记录
- 如需处理更多，请分批操作

### Q5: 创建账户后可以修改账户名称吗？

**A**: **不可以**。账户名称是账户的唯一标识，创建后不可修改。

| 项目 | 说明 |
|------|------|
| **原因** | 账户名称与系统内部多处数据关联<br>修改名称会导致数据关联错误<br>为保证数据一致性，系统禁止修改 |
| **建议** | 创建前仔细确认账户名称<br>使用有意义且不易混淆的命名<br>如果确实需要修改，只能删除重建（会丢失历史数据）|
```

## 4. 编写注意事项

### 4.1 好的实践（DO ✅）

| 实践 | 说明 |
|------|------|
| **明确的步骤编号** | 每个步骤清晰标注<br>步骤之间逻辑连贯<br>用户可以按照步骤一步步操作 |
| **详细的截图占位符** | 标明需要补充截图的位置<br>说明截图应该包含的内容<br>提供截图命名规范 |
| **常见错误处理** | 列出可能遇到的错误<br>提供明确的解决方法<br>帮助用户自助解决问题 |
| **友好的语言** | 避免技术术语<br>使用用户能理解的表达<br>语气亲切，便于阅读 |
| **多种帮助途径** | 在线客服<br>邮箱支持<br>电话支持<br>相关文档链接 |

### 4.2 禁止的做法（DON'T ❌）

| 禁止项 | 说明 |
|--------|------|
| ❌ **使用模糊描述** | 禁止："点击相关按钮"<br>应该："点击页面右上角的'新建账户'按钮" |
| ❌ **缺少异常说明** | 必须说明可能遇到的错误<br>必须提供解决方法 |
| ❌ **步骤跳跃** | 不能省略中间步骤<br>每个操作都要说明清楚 |
| ❌ **没有实际截图** | 不能长期依赖占位符<br>必须在规定时间内补充截图 |
| ❌ **缺少 FAQ** | 必须包含至少 5-8 个常见问题<br>基于真实用户反馈整理 |
