## 📝 文档命名规范

- 格式：`PRD_{功能名称}.md`
- 示例：`PRD_账户开通管理.md`、`PRD_素材批量上传.md`

## 📖 文档结构

每个 PRD 文档包含以下核心章节：

1. **文档信息**：功能点 ID、版本、日期等
2. **背景与目标**：业务背景、目标用户、核心目标
3. **用户故事**：具体的用户使用场景
4. **功能需求**：详细的功能描述
5. **交互流程**：主流程和异常流程
6. **UI 原型**：页面结构和字段定义
7. **数据定义**：API 接口和数据模型
8. **业务规则**：校验规则、权限规则、状态流转
9. **非功能需求**：性能、兼容性、可用性要求
10. **验收标准**：功能、UI、性能验收条件
11. **变更记录**：版本更新历史

## 🎯 使用指南

### 创建新 PRD

使用以下 Cursor Prompt 创建新 PRD：

```
创建新功能：[功能名称]

功能点 ID: [自动生成或手动指定]
所属模块: [模块名称]
功能背景: [功能背景说明]
用户角色: [目标用户]
核心场景: [使用场景]

请生成：
1. PRD 文档（按 prd-standards.md 规范）
2. 前端原型代码（按 frontend-standards.md 规范）
3. 用户操作手册（按 user-manual-standards.md 规范）
```

### 更新现有 PRD

在对功能进行修改时，务必同步更新 PRD 文档中的：

- 功能需求描述
- 交互流程
- 验收标准
- 变更记录（新增版本记录）
